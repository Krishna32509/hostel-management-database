#Hostel Room Allocation & Complaint Management System

A fully normalized database system (up to 5NF) built using MySQL 8.x to manage hostel room allocations and student complaints efficiently. The project focuses on backend design, data integrity, and automation using SQL and procedural extensions.

⸻

🚀 Features

* Room Allocation Management
    * Assign students to rooms with capacity enforcement
    * Prevent over-allocation using triggers
    * Ensure gender-based hostel constraints
* Complaint Management System
    * Register, track, and resolve complaints
    * Categorized complaint types (Electrical, Plumbing, Network, etc.)
    * Real-time status updates (Pending → In Progress → Resolved)
* Data Integrity & Constraints
    * Primary keys, foreign keys, UNIQUE & CHECK constraints
    * Fully normalized schema (up to 5NF)
* Automation with Triggers
    * Prevent room over-capacity
    * Enforce gender-hostel consistency
* Stored Procedures
    * Register complaints
    * Vacate students
    * Resolve complaints with validation
* Advanced SQL Queries
    * Joins, subqueries, aggregation, GROUP BY, HAVING
    * Occupancy reports and complaint analytics
